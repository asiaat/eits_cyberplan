# Domain Glossary

| Term | Meaning |
|------|---------|
| E-ITS | Estonian Information Security Standard |
| ISMS | Information Security Management System |
| IMR | Information security Measures Implementation Plan |
| Tenant | Organization using the system |
| Business Process | Organizational process that needs protection |
| Asset | Information, system, service, component, location requiring protection |
| Target Object | Object to which E-ITS modules/measures may apply |
| Protection Need | Required level of confidentiality, integrity, availability |
| Module | E-ITS baseline security module |
| Measure | E-ITS security measure |
| Evidence | Document, file, URL proving implementation |
| Audit Log | Immutable change history |
| Risk Treatment | Decision on how to handle a risk |

## Protection Need Values

- `normal` (normaalne)
- `high` (suur)
- `very_high` (väga suur)
- `unknown` (määramata)

## IMR Status Values

- `not_started`
- `in_progress`
- `implemented`
- `not_applicable`
- `accepted_exception`
- `needs_review`

## Risk Treatment Values

- `mitigate` (reduce)
- `accept`
- `avoid`
- `transfer`
